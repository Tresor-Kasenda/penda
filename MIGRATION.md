## Future Template

When breaking changes occur, document them using this structure:

### Changed
- What changed (API/signature/behavior)

### Why
- Why the change was needed

### Before
```go
// old code
```

### After
```go
// new code
```

### Notes
- edge cases
- compatibility caveats
